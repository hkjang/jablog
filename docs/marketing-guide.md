# 마케팅 연동 가이드

## 개요

JaBlog는 소셜 미디어 자동 공유 및 이메일 발송 기능을 통해 마케팅 자동화를 지원합니다. 블로그 콘텐츠 발행과 동시에 다양한 채널로 홍보할 수 있습니다.

---

## 지원 플랫폼

| 플랫폼      | 유형     | 기능           | 연동 상태 |
| ----------- | -------- | -------------- | --------- |
| WordPress   | 블로그   | 포스트 발행    | ✅ 지원   |
| Tistory     | 블로그   | 포스트 발행    | ✅ 지원   |
| 소셜 미디어 | 공유     | 발행 알림 공유 | 🔜 예정   |
| 이메일      | 뉴스레터 | 발행 알림 발송 | 🔜 예정   |

## 플랫폼 연결 설정

### 단계 1: 설정 페이지 접근

1. 좌측 사이드바에서 **설정** 메뉴를 클릭합니다.
2. **플랫폼 연동** 탭을 선택합니다.

---

### 단계 2: WordPress 연결

#### 사전 준비

| 항목             | 설명             | 확인 방법          |
| ---------------- | ---------------- | ------------------ |
| WordPress 사이트 | 설치 및 운영 중  | URL 접속 확인      |
| 관리자 권한      | 편집자 이상 권한 | 사용자 역할 확인   |
| REST API         | 활성화 상태      | /wp-json/ 접속     |
| 앱 비밀번호      | 생성 가능        | 프로필 페이지 확인 |

#### 앱 비밀번호 생성

1. WordPress 관리자 페이지에 로그인합니다.
2. **사용자** → **프로필**로 이동합니다.
3. **애플리케이션 비밀번호** 섹션을 찾습니다.
4. 새 애플리케이션 이름을 입력합니다 (예: "JaBlog").
5. **새 애플리케이션 비밀번호 추가** 버튼을 클릭합니다.
6. 생성된 비밀번호를 복사합니다 (한 번만 표시됨).

#### JaBlog에 등록

1. **설정** → **플랫폼 연동**으로 이동합니다.
2. **새 플랫폼 연동 추가**를 클릭합니다.
3. 플랫폼으로 **WordPress**를 선택합니다.
4. 필수 정보를 입력합니다:

| 필드        | 입력값          | 예시               |
| ----------- | --------------- | ------------------ |
| 이름        | 표시용 이름     | 내 워드프레스      |
| 사이트 URL  | WordPress 주소  | https://myblog.com |
| 사용자명    | 로그인 아이디   | admin              |
| 앱 비밀번호 | 생성한 비밀번호 | xxxx xxxx xxxx     |

5. **추가** 버튼을 클릭합니다.
6. **테스트** 버튼으로 연결을 확인합니다.

---

### 단계 3: Tistory 연결

#### 사전 준비

| 항목           | 설명             | 확인 방법        |
| -------------- | ---------------- | ---------------- |
| Tistory 블로그 | 블로그 개설 완료 | 블로그 접속      |
| Open API 앱    | 앱 등록 완료     | 개발자 센터 확인 |
| Access Token   | 토큰 발급 완료   | OAuth 인증       |

#### API 앱 등록 및 토큰 발급

1. [Tistory Open API](https://www.tistory.com/guide/api/manage/register)에 접속합니다.
2. 새 애플리케이션을 등록합니다:

| 항목         | 입력값                         |
| ------------ | ------------------------------ |
| 서비스명     | JaBlog                         |
| 서비스 URL   | http://localhost:3000          |
| Callback URL | http://localhost:3000/callback |

3. Client ID와 Secret Key를 확인합니다.
4. OAuth 인증을 통해 Access Token을 발급받습니다.

#### JaBlog에 등록

1. **설정** → **플랫폼 연동**으로 이동합니다.
2. **새 플랫폼 연동 추가**를 클릭합니다.
3. 플랫폼으로 **Tistory**를 선택합니다.
4. 필수 정보를 입력합니다:

| 필드         | 입력값        | 예시        |
| ------------ | ------------- | ----------- |
| 이름         | 표시용 이름   | 내 티스토리 |
| 블로그명     | URL 식별자    | myblog      |
| Access Token | 발급받은 토큰 | xxxxxxxx    |

5. **추가** 버튼을 클릭합니다.
6. **테스트** 버튼으로 연결을 확인합니다.

---

### 단계 4: 권한 승인

#### WordPress 권한

| 권한          | 필요 역할   | 설명          |
| ------------- | ----------- | ------------- |
| 포스트 작성   | 작성자 이상 | 새 글 발행    |
| 포스트 수정   | 편집자 이상 | 글 수정       |
| 미디어 업로드 | 작성자 이상 | 이미지 첨부   |
| 카테고리 관리 | 편집자 이상 | 카테고리 지정 |

#### Tistory 권한

| 권한             | OAuth Scope | 설명           |
| ---------------- | ----------- | -------------- |
| 블로그 정보 읽기 | blog        | 기본 정보 조회 |
| 글 작성          | write       | 포스트 발행    |
| 글 수정          | write       | 포스트 수정    |

## 연동 관리

### 연동 정보 수정

1. 등록된 플랫폼 카드에서 **수정** 버튼을 클릭합니다.
2. 변경할 정보를 수정합니다.
3. 토큰/비밀번호 변경 시 새 값을 입력합니다.
4. **저장** 버튼을 클릭합니다.

### 연결 테스트

1. 플랫폼 카드에서 **테스트** 버튼을 클릭합니다.
2. 연결 성공 시 "연결 성공!" 메시지가 표시됩니다.
3. 실패 시 오류 메시지를 확인하고 설정을 수정합니다.

### 연동 삭제

1. 플랫폼 카드에서 **삭제** 버튼을 클릭합니다.
2. 확인 대화상자에서 **확인**을 클릭합니다.
3. 해당 플랫폼 연동이 제거됩니다.

## 문제 해결

### WordPress 연결 오류

| 오류 메시지       | 원인          | 해결 방법              |
| ----------------- | ------------- | ---------------------- |
| 401 Unauthorized  | 인증 실패     | 사용자명/비밀번호 확인 |
| 403 Forbidden     | 권한 부족     | 사용자 역할 확인       |
| 404 Not Found     | 잘못된 URL    | 사이트 URL 확인        |
| Connection Failed | 네트워크 오류 | 사이트 접속 확인       |

### Tistory 연결 오류

| 오류 메시지   | 원인           | 해결 방법          |
| ------------- | -------------- | ------------------ |
| Token Expired | 토큰 만료      | 새 토큰 발급       |
| Invalid Blog  | 블로그명 오류  | 블로그 식별자 확인 |
| API Limit     | 호출 한도 초과 | 다음 날 재시도     |
