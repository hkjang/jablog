# 오류 해결 가이드

## 개요

JaBlog 사용 중 발생할 수 있는 오류 목록과 해결 방법을 제공합니다. 각 오류 유형별로 원인, 해결 방법, 예방 팁을 정리했습니다.

---

## 연결 오류

### 플랫폼 연결 실패

| 오류                | 원인               | 해결 방법                          | 예방 팁                        |
| ------------------- | ------------------ | ---------------------------------- | ------------------------------ |
| WordPress 연결 실패 | 잘못된 인증 정보   | 사용자명, 앱 비밀번호 재확인       | 앱 비밀번호 안전하게 저장      |
| 401 Unauthorized    | 비밀번호 만료/변경 | 새 앱 비밀번호 생성 후 재등록      | 비밀번호 변경 시 즉시 업데이트 |
| 403 Forbidden       | 권한 부족          | 사용자 역할을 편집자 이상으로 변경 | 충분한 권한의 계정 사용        |
| 404 Not Found       | 잘못된 URL         | https:// 포함 정확한 URL 입력      | URL 복사/붙여넣기 사용         |
| Tistory 연결 실패   | Access Token 만료  | 새 토큰 발급 후 재등록             | 토큰 만료일 확인               |
| Invalid Blog        | 블로그명 오류      | URL에서 식별자 정확히 확인         | myblog.tistory.com → "myblog"  |

### API 연결 오류

| 오류                  | 원인             | 해결 방법                   | 예방 팁                |
| --------------------- | ---------------- | --------------------------- | ---------------------- |
| Connection Timeout    | 서버 응답 없음   | 대상 서버 상태 확인, 재시도 | 네트워크 상태 확인     |
| SSL Certificate Error | SSL 인증서 문제  | 사이트 SSL 설정 확인        | 유효한 SSL 인증서 사용 |
| Network Error         | 인터넷 연결 문제 | 네트워크 연결 확인          | 안정적인 네트워크 환경 |

## 오류 확인 방법

### 1. API 모니터링

1. 좌측 메뉴에서 **API 모니터링**을 클릭합니다.
2. 오류가 발생한 API 호출을 확인합니다.
3. 오류 코드와 메시지를 확인합니다.

### 2. 브라우저 개발자 도구

1. **F12** 또는 **Ctrl+Shift+I**를 누릅니다.
2. **Console** 탭에서 오류 메시지를 확인합니다.
3. **Network** 탭에서 실패한 요청을 확인합니다.

### 3. 서버 로그

1. 백엔드 서버 터미널을 확인합니다.
2. 오류 메시지와 스택 트레이스를 확인합니다.

## 오류 코드 참조

| 코드 | 의미             | 일반적 조치                 |
| ---- | ---------------- | --------------------------- |
| 400  | 잘못된 요청      | 입력 데이터 확인            |
| 401  | 인증 필요        | 로그인 또는 토큰 확인       |
| 403  | 권한 없음        | 사용자 권한 확인            |
| 404  | 찾을 수 없음     | URL 또는 리소스 확인        |
| 500  | 서버 오류        | 서버 로그 확인, 관리자 문의 |
| 502  | 게이트웨이 오류  | 서버 상태 확인              |
| 503  | 서비스 이용 불가 | 서버 재시작 필요            |

오류가 계속되면 다음 정보와 함께 관리자에게 문의하세요:

2. **발생 시간**
3. **수행하던 작업**
4. **브라우저 정보**
5. **스크린샷** (가능한 경우)
